这份接口文档基于 RESTful 风格设计。为了保证“易拓展”和前后端分离的规范，我们将接口严格分为两大类：**公共接口（前台展示使用，免登录）** 和 **管理接口（后台管理使用，需校验 JWT Token）**。

---

### 零、 全局统一定义

#### 1. 统一响应格式 (Result<T>)
后端所有的接口均返回这个 JSON 结构，前端可以通过响应拦截器统一处理。
```json
{
  "code": 200,         // 状态码：200成功，401未登录，403无权限，500服务器错误
  "message": "操作成功", // 提示信息
  "data": { ... }      // 实际的数据，可能是对象或数组
}
```

#### 2. 分页请求与响应规范
列表请求通常带有分页参数 `page` (当前页, 默认1) 和 `size` (每页条数, 默认10)。
分页响应的 `data` 结构通常如下：
```json
"data": {
  "total": 100,      // 总条数
  "records": [ ... ] // 当前页的数据列表
}
```

### 一、 开放接口 API (前台展示端 `/api/public`)
> 💡 特点：无需携带 Token，直接请求。文章列表仅返回 `status = 1` (已发布) 且分类 `is_visible = 1` 的数据。

#### 1. 获取顶部分类列表
* **接口路径**: `GET /api/public/categories`
* **接口描述**: 前台展示上边栏导航，按 `sort_order` 排序。
* **响应 Data**: 
  ```json
  [
    { "id": 1, "name": "后端技术" },
    { "id": 2, "name": "前端技术" }
  ]
  ```

#### 2. 获取右侧标签列表
* **接口路径**: `GET /api/public/tags`
* **接口描述**: 前台展示右侧标签栏。
* **响应 Data**: 
  ```json
  [
    { "id": 1, "name": "Spring Boot" },
    { "id": 2, "name": "Vue3" }
  ]
  ```

#### 3. 分页获取文章列表 (支持多条件筛选)
* **接口路径**: `GET /api/public/articles`
* **请求参数 (Query)**:
  * `page`: 当前页 (默认 1)
  * `size`: 每页数量 (默认 10)
  * `categoryId`: (可选) 按分类筛选
  * `tagId`: (可选) 按标签筛选
  * `keyword`: (可选) 标题或摘要搜索
* **响应 Data**:
  ```json
  {
    "total": 50,
    "records": [
      {
        "id": 101,
        "title": "SpringBoot整合JWT教程",
        "summary": "本文介绍了如何在SpringBoot项目中...",
        "viewCount": 120,
        "createTime": "2024-05-10 10:00:00",
        "categoryName": "后端技术",
        "tags": ["Spring Boot", "JWT"]
      }
    ]
  }
  ```

#### 4. 获取文章详情
* **接口路径**: `GET /api/public/articles/{id}`
* **接口描述**: 根据 ID 获取文章完整 Markdown 内容，并在后端将该文章的 `view_count` +1。
* **响应 Data**:
  ```json
  {
    "id": 101,
    "title": "SpringBoot整合JWT教程",
    "content": "# 这里是 Markdown 原文...",
    "categoryName": "后端技术",
    "viewCount": 121,
    "createTime": "2024-05-10 10:00:00",
    "tags": [
      {"id": 1, "name": "Spring Boot"}
    ]
  }
  ```

### 二、 后台管理 API (后台管理端 `/api/admin`)
> 💡 特点：除登录接口外，所有请求头必须携带 `Authorization: Bearer <Token>`。

#### 1. 管理员登录
* **接口路径**: `POST /api/admin/login`
* **请求参数 (Body/JSON)**:
  ```json
  {
    "username": "admin",
    "password": "yourpassword"
  }
  ```
* **响应 Data**:
  ```json
  {
    "token": "eyJhbGciOiJIUzUxMiJ9...",
    "user": {
      "nickname": "管理员",
      "avatar": "xxx.jpg"
    }
  }
  ```

---

#### 2. 分类管理 (Category)
* **获取分类列表 (含隐藏的)**: `GET /api/admin/categories` (支持分页)
* **新增分类**: `POST /api/admin/categories`
  * Body: `{ "name": "新分类", "description": "...", "sortOrder": 1, "isVisible": 1 }`
* **修改分类**: `PUT /api/admin/categories`
  * Body: `{ "id": 1, "name": "修改后的分类", ... }`
* **删除分类**: `DELETE /api/admin/categories/{id}`

---

#### 3. 标签管理 (Tag)
* **获取标签列表**: `GET /api/admin/tags` (支持分页)
* **新增标签**: `POST /api/admin/tags`
  * Body: `{ "name": "MySQL" }`
* **修改标签**: `PUT /api/admin/tags`
* **删除标签**: `DELETE /api/admin/tags/{id}`

---

#### 4. 文章管理 (Article)
* **获取后台文章列表**: `GET /api/admin/articles`
  * *参数*: `page`, `size`, `title`, `status` (包含草稿)
* **新增发布/存草稿**: `POST /api/admin/articles`
  * Body: 
    ```json
    {
      "title": "新文章",
      "summary": "摘要",
      "content": "正文...",
      "categoryId": 1,
      "tagIds": [1, 2],
      "status": 1   // 1发布, 0草稿
    }
    ```
* **修改文章**: `PUT /api/admin/articles`
* **删除文章**: `DELETE /api/admin/articles/{id}`
* **获取文章详情(用于回显编辑)**: `GET /api/admin/articles/{id}`

---

#### 5. 文件与资源管理 (File)
* **上传文件 (核心)**: `POST /api/admin/files/upload`
  * **请求类型**: `multipart/form-data`
  * **参数**: `file` (文件对象)
  * **响应 Data**: 
    ```json
    {
      "fileName": "avatar.png",
      "fileUrl": "/uploads/2024/05/uuid.png",
      "fileType": "png",
      "fileSize": 102400
    }
    ```
* **获取文件列表**: `GET /api/admin/files`
  * *参数*: `page`, `size`
* **删除文件记录**: `DELETE /api/admin/files/{id}`
  * *(注：可以根据需要决定是否同时删除服务器硬盘上的物理文件)*

### 三、 前端对接建议（Axios 封装）

在前端 Vue 项目中，强烈建议将 `axios` 封装一下，自动处理 JWT Token 和全局错误：

```javascript
// src/utils/request.js
import axios from 'axios'

const service = axios.create({
  baseURL: 'http://localhost:8080', // 后端地址
  timeout: 5000
})

// 请求拦截器：如果访问后台接口，自动带上 token
service.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token && config.url.includes('/api/admin')) {
    config.headers['Authorization'] = 'Bearer ' + token
  }
  return config
})

// 响应拦截器：统一处理报错
service.interceptors.response.use(res => {
  const data = res.data
  if (data.code !== 200) {
    // 处理各种报错情况...例如 token 过期跳回登录页
    if (data.code === 401) {
      window.location.href = '/admin/login'
    }
    return Promise.reject(new Error(data.message || 'Error'))
  }
  return data.data // 直接返回实际数据，组件里不用写 res.data.data
})

export default service
```

这份文档设计满足了你所需要的分类、标签、文章管理以及文件管理（上传文章附件）的需求，同时极大地降低了前后台耦合。
